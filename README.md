# GraphProcessor

## Что это такое?
Платформа для построения графовых маршрутов, поиска кратчайших путей и вычисления расстояний между вершинами.

## Зачем нужно?
Проект в первую очередь был направлен на визуализацию работы графовых алгоритмов, таких как "Поиск в глубину(DFS)", "Поиск в ширину(BFS)", "Алгоритм Дейкстры(Dijkstra)".\
Приложение имеет потенциал для интеграции бизнес-логики, такой как система организации транспортной логистики(для e-commerce корпораций) или для проектирования рекомендательных алгоритмов(поисковые движки).

## Технологический стек
- Backend🧠 : C#, ASP.NET Entity Framework, xUnit
- Data💾 : PostgreSQL, Redis, S3-compatible(MinIO)
- Frontend📱 : TypeScript, Vue3, vis.js(vis-netowrk)
- Infrastructure🏢 : Docker, Docker-Compose, Nginx, Github Actions(CI/CD)

## Архитектура:
### Backend:
- Models: Объекты сущностей EF Core, DTO для типизации и использования данных между слоями сервисов
- Controllers: API контроллеры ASP.NET
- Services: Сервисы с бизнес-логикой приложения(интегрируются через DI контейнеры как зависимости)
- Data: Контекст базы данных для EF
- Migrations: Миграции базы данных
### Frontend:
- components: UI компоненты Vue
- models: Типы и интерфейсы для надежной типизации данных.
- views: Представления Vue, привязанные к SPA-роутеру
- router: Организация SPA-роутинга
- services: Сервсисы для независимой от компонентов интерфейса бизнес логики

### Какие задачи были решены?
1. Изучены и реализованны фундаментальные алгоритмы обхода графов: DFS: для полного обхода графа, BFS: для эффективного поиска кратчейшего пути в графе. Dijkstra: для поиска кратчайшего пути в взвешенном графе
2. Спроектирован API на языке C# с использованием фреймворка ASP NET.
3. Спроектирована клиентская часть приложения на компонентном фреймворке Vue.js.
4. Интегрирована библиотека vis.js в компонентно-реактивную среду Vue. Реализована логика добавления/удаления узлов в граф, предоставленна возможность выбора вида графа
5. Соблюдены архитектурные принципы(разделение ответственности, слои, внедрение зависимостей
6. Спроектированна база данных postgresql, интегрирована в ORM EF Core через скаффолдинг
7. Реализованная JWT аутентификация через access токены и Claims
8. Настроен базовый DevOps и деплой: docker-compose и docker с multi-stage сборкой, настроен веб-сервер nginx с отдачи статических файлов и проксированием Kestrel
9. Алгоритмы погрыты тестами
